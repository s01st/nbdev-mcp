# nbdev MCP VS Code Extension

VS Code extension for nbdev projects with MCP server integration.

## Features

### Project Management
- **Set Active Project**: Select an nbdev project folder
- **Show Current Project**: Display info about the active project
- Auto-detection of nbdev projects in workspace

### nbdev Workflow Commands
- **Export Notebooks**: Run `nbdev_export`
- **Prepare**: Run `nbdev_prepare` (export, test, clean)
- **Run Tests**: Execute notebook tests

### Linting
- **Lint Notebooks**: Check for nbdev best practices
- **Check Unused Imports**: Find unused imports in notebook cells
- Diagnostics displayed in Problems panel
- Optional lint-on-save

### Analysis Tools
- **Find Symbol**: Locate where a symbol is exported
- **Analyze Exports**: See what a notebook exports
- **Dependency Tree**: Visualize module dependencies
- **Audit Module Index**: Check _modidx.py for issues

### Error Detection
- **Scan Notebook Errors**: Find existing error outputs
- **Run Tutorials**: Execute tutorial notebooks

### Source Tracking
- **Find Source Notebook**: Locate the notebook that generates a .py file
- **Check if Generated**: See if a file is auto-generated by nbdev

## Installation

### From Source

1. Clone this repository
2. Install dependencies:
   ```bash
   cd vscode-extension
   npm install
   ```
3. Compile:
   ```bash
   npm run compile
   ```
4. Press F5 in VS Code to launch Extension Development Host

### Package as VSIX

```bash
npm run package
```

Then install the generated `.vsix` file via VS Code.

## Configuration

| Setting | Default | Description |
|---------|---------|-------------|
| `nbdev-mcp.mcpServerPath` | `""` | Path to Python (uses system PATH if empty) |
| `nbdev-mcp.autoDetectProject` | `true` | Auto-detect nbdev project from workspace |
| `nbdev-mcp.showDiagnostics` | `true` | Show lint diagnostics in Problems panel |
| `nbdev-mcp.lintOnSave` | `false` | Run lint when saving notebooks |

## Requirements

- VS Code 1.85.0 or later
- Python with `nbdev_mcp` package installed:
  ```bash
  pip install nbdev-mcp
  ```

## Commands

All commands are prefixed with `nbdev:` in the command palette.

| Command | Description |
|---------|-------------|
| `nbdev: Set Active Project` | Select project folder |
| `nbdev: Show Current Project` | Display project info |
| `nbdev: Export Notebooks` | Run nbdev_export |
| `nbdev: Prepare` | Run nbdev_prepare |
| `nbdev: Run Tests` | Execute tests |
| `nbdev: Lint Notebooks` | Check best practices |
| `nbdev: Check Unused Imports` | Find unused imports |
| `nbdev: Find Symbol` | Locate symbol exports |
| `nbdev: Analyze Notebook Exports` | Show notebook exports |
| `nbdev: Show Dependency Tree` | Visualize dependencies |
| `nbdev: Audit Module Index` | Check _modidx.py |
| `nbdev: Scan Notebook Errors` | Find error outputs |
| `nbdev: Run Tutorials` | Execute tutorials |
| `nbdev: Find Source Notebook` | Find .py source |
| `nbdev: Check if Generated` | Check if auto-generated |

## Context Menu

Right-click on files for quick access:

- **On .py files**: "Find Source Notebook", "Check if Generated"
- **On .ipynb files**: "Analyze Notebook Exports"
