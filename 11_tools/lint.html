 lint_rules (project:Optional[str]=None, fix_relative:bool=False)

lint_imports

 lint_imports (project:Optional[str]=None,
               ignore_patterns:Optional[List[str]]=None)

Check for unused imports in notebook cells.

Analyzes each code cell to find imports that are never used within that cell. Uses AST parsing for accurate detection.

Type Default Details
project Optional None Project path or alias. Uses current project if not specified.
ignore_patterns Optional None List of import names to ignore (e.g., [‘typing’, ‘annotations’]).
Returns Dict Result with ‘issues’ list of unused imports per cell.

lint_types

 lint_types (project:Optional[str]=None, strict:bool=False)

Check type annotations in notebook cells.

Validates that exported functions and classes have type hints. Uses AST parsing to detect missing annotations.

Type Default Details
project Optional None Project path or alias. Uses current project if not specified.
strict bool False If True, require all parameters to have type hints.
If False (default), only check return type annotations.
Returns Dict Result with ‘issues’ list of missing type annotations.

auto_fix_lint

 auto_fix_lint (project:Optional[str]=None, fix_relative:bool=True,
                fix_inits:bool=True, dry_run:bool=False)

Auto-fix common lint issues in notebooks.

Automatically fixes: - Relative imports -> absolute imports - Incorrect default_exp in 00__init__.ipynb notebooks

Type Default Details
project Optional None Project path or alias. Uses current project if not specified.
fix_relative bool True If True, convert relative imports to absolute.
fix_inits bool True If True, fix incorrect default_exp in init notebooks.
dry_run bool False If True, report what would be fixed without making changes.
Returns Dict Result with ‘fixes’ list of applied/proposed fixes.

lint_notebook_structure

 lint_notebook_structure (project:Optional[str]=None, fix:bool=False)

Check and fix notebook structure for nbdev conventions.

Checks that notebooks have: - Standard ending structure (## Next, empty export, ## Export, nbdev_export) - Properly formatted headers with backticks for code names

Type Default Details
project Optional None Project path or alias. Uses current project if not specified.
fix bool False If True, automatically fix structure issues.
Returns Dict Result dict with:
- ok: bool - Success status
- issues: List[Dict] - Structure issues found
- fixed: int - Number of notebooks fixed (if fix=True)
- pretty: str - Formatted output

lint_private_attributes

 lint_private_attributes (project:Optional[str]=None,
                          allow_patterns:Optional[List[str]]=None)

Flag private attributes (self._xxx) in exported classes.

MCP tools should be stateless. Private attributes suggest hidden state that persists between calls, which breaks the MCP model.

Type Default Details
project Optional None Project path or alias.
allow_patterns Optional None List of attribute name patterns to allow (e.g., [’_cache’, ’_lock’]).
Returns Dict Result with ‘issues’ list of private attribute usages.

lint_cell_complexity

 lint_cell_complexity (project:Optional[str]=None, max_classes:int=1,
                       max_functions:int=2)

Flag cells with too many definitions (classes/functions).

nbdev encourages one class or function per cell for: - Granular testing - Clear cell-level documentation - Literate programming readability

Type Default Details
project Optional None Project path or alias.
max_classes int 1 Maximum classes allowed per cell (default: 1).
max_functions int 2 Maximum functions allowed per cell (default: 2).
Returns Dict Result with ‘issues’ list of overly complex cells.

source

lint_import_order

 lint_import_order (project:Optional[str]=None)

Check that from __future__ import is in the first export cell.

Python requires future imports to be at the top of a module. In nbdev, this means the first export cell must contain any future imports, before other export cells.

Type Default Details
project Optional None Project path or alias.
Returns Dict Result with ‘issues’ list of import order violations.
