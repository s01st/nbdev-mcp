analyze_dependency_order

 analyze_dependency_order (project:Optional[str]=None, enforce:bool=False)

Analyze notebook dependency graph to suggest numbering order.

Ranks notebooks by their internal import dependencies to determine the optimal ordering. Optionally enforces the suggested numbering by renaming notebooks.

Type Default Details
project Optional None Project path or alias. Uses current project if not specified.
enforce bool False If True, rename notebooks to match suggested order.
Returns Dict Result with ‘suggestions’ list and ‘applied’ count if enforce=True.

dependency_tree

 dependency_tree (project:Optional[str]=None, scope:str='internal',
                  include_unused:Optional[str]=None,
                  write_qmd:Optional[str]=None)

Summarize internal and external module dependencies.

Analyzes import statements across all notebooks to build a dependency graph. Can generate Mermaid and DOT diagrams.

Type Default Details

scope str internal ‘internal’, ‘external’, or ‘both’ to control which deps to show.
include_unused Optional None ‘internal’, ‘external’, or ‘both’ to include unused imports.
write_qmd Optional None Path to write a Quarto document with diagrams.
Returns Dict Result with nodes, edges, mermaid/dot diagrams, and optional written file.

 lookup_external_symbol (symbol:str, packages:Optional[List[str]]=None,

interactive_dependency_graph

interactive_dependency_graph

 interactive_dependency_graph (project:Optional[str]=None,
                               output_path:Optional[str]=None,
                               include_duplicates:bool=True,
                               similarity_threshold:float=0.8,
                               open_browser:bool=True)

Generate an interactive hierarchical dependency graph.

Creates a Cytoscape.js visualization showing: - Module level (top): Package modules as containers - Notebook level (middle): Notebooks within each module - Symbol level (bottom): Functions, classes, constants within notebooks

Edges show: - Import dependencies (A supports B if B imports from A) - Code duplication (exact, similar, suspected)

Type Default Details
project Optional None Project path or alias. Uses current project if not specified.
output_path Optional None Output HTML path. Defaults to project/dep_graph.html.
include_duplicates bool True Whether to detect and show code duplications.
similarity_threshold float 0.8 Minimum similarity for ‘similar’ duplicates (0.5-1.0).
open_browser bool True Whether to open the graph in a browser.
Returns Dict Result dict with:
- ok: bool - Success status
- output_path: str - Path to generated HTML file
- stats: Dict - Graph statistics (modules, notebooks, symbols, edges)
- pretty: str - Formatted output summary
